# DinoDoo 产品方案 V0.1

## 一句话定位

DinoDoo 是一个给 3 岁宝宝玩的卡通恐龙 Agent 应用：宝宝用语音和恐龙朋友团一起讲故事、做选择、生成恐龙卡片和小漫画，家长负责设置主题、时长和安全边界。

## 目标用户

- 宝宝：3 岁左右，喜欢恐龙、图像、声音、重复节奏和简单选择。
- 家长：希望孩子玩得开心，但内容安全、低刺激、可控，不希望孩子沉迷或接触开放社交。

## 核心体验

DinoDoo 不应该像普通聊天机器人，而应该像一个会回应宝宝的“恐龙小剧场”。

核心循环：

1. 家长选择主题，例如睡前、冒险、学颜色、学数字、情绪表达。
2. 恐龙主动开场，用一句很短的剧情邀请宝宝参与。
3. 宝宝用语音回答或选择。
4. 恐龙根据宝宝的话推进故事，并给两个简单选项。
5. 每次游戏结束生成一张“今日恐龙卡片”或一页小漫画。
6. 家长可以保存、重放或分享给家人。

## MVP 范围

第一版建议叫“恐龙小剧场”。

必须有：

- 3 只固定恐龙角色。
- 语音输入。
- 语音回复。
- 每轮 1 到 3 分钟的短故事互动。
- 4 个主题：睡前、冒险、学颜色、学数字。
- 每轮结束生成 1 张恐龙卡片。
- 家长控制：每日时长、主题、是否允许生成图片、内容记录开关。

先不做：

- 开放式多用户社交。
- 复杂长期养成系统。
- 真实恐龙科普数据库。
- 很复杂的多 Agent 编排。
- 宝宝自由浏览社区内容。

## 恐龙角色设定

### 小暴

- 原型：小霸王龙。
- 性格：勇敢、好奇、但不凶。
- 口头禅：小脚丫，出发啦！
- 适合场景：冒险、选择路线、找东西。

### 花花

- 原型：三角龙。
- 性格：温柔、会照顾朋友。
- 口头禅：慢慢来，我们一起想办法。
- 适合场景：睡前、情绪表达、礼貌用语。

### 咕噜

- 原型：腕龙。
- 性格：慢吞吞、爱吃水果、声音软。
- 口头禅：咕噜咕噜，我看到啦。
- 适合场景：颜色、数字、找不同。

MVP 技术上可以先用一个 Agent 模拟多个角色，不必一开始做真正的多 Agent 系统。

## 互动规则

面向 3 岁宝宝，互动要非常短、清楚、有节奏。

- 每次只问一个问题。
- 每次最多给两个选择。
- 单句尽量不超过 15 个汉字。
- 多用重复句式，例如“找一找，看一看，摸摸小脑袋”。
- 不讲恐怖、捕食、死亡、受伤、打斗。
- 不纠正宝宝太多，优先鼓励和顺着想象走。
- 当宝宝沉默时，恐龙主动给简单选择。

示例：

> 小暴：森林里有一颗蓝色蛋！我们去看看吗？
>
> 宝宝：去！
>
> 花花：好呀。我们走小路，还是走彩虹桥？

## 生成内容

第一版只做三种作品中的一种，建议先做“恐龙卡片”。

### 恐龙卡片

- 一张竖版卡片。
- 包含恐龙角色、宝宝选择的颜色、一个动作、一个小物品。
- 家长可保存到相册。

示例指令：

- 蓝色小霸王龙吃草莓。
- 花花戴黄色帽子。
- 咕噜在彩虹桥旁边数星星。

### 后续扩展

- 表情动作：开心、困困、抱抱、挥手。
- 四格漫画：今天的故事回顾。
- 睡前故事页：一张图加一句晚安话。

## 家长模式

家长模式是这个产品能不能放心用的关键。

第一版需要：

- 设置每日使用时长。
- 选择可用主题。
- 开关图片生成。
- 查看今天生成的卡片。
- 删除记录。
- 关闭个性化记忆。

默认策略：

- 不开放陌生人社交。
- 不展示公开内容流。
- 不让宝宝输入个人信息。
- 不把宝宝语音长期保存为默认行为。
- 家长确认后才保存作品。

## 内容安全边界

系统提示词和内容策略要明确：

- 恐龙永远友善、温柔、卡通化。
- 不描述血腥、伤害、死亡、捕猎。
- 不引导孩子离开家长视线。
- 不收集姓名、地址、学校、电话等信息。
- 不对孩子做心理诊断或医学建议。
- 遇到危险、身体不舒服、害怕等内容时，温柔提醒找家长。

## 产品结构

建议第一版只有三个主要界面：

1. 宝宝游玩页
   - 大恐龙形象。
   - 语音按钮。
   - 两个大选择按钮。
   - 简单动画反馈。

2. 今日作品页
   - 恐龙卡片列表。
   - 保存、删除、重放故事。

3. 家长设置页
   - 主题。
   - 时长。
   - 生成开关。
   - 隐私与记录。

## 技术雏形

MVP 可以这样拆：

- 前端：移动端优先的 Web App 或小程序式界面。
- 语音输入：浏览器语音识别或移动端语音能力。
- 语音输出：TTS，优先选择温柔、童话感声音。
- 对话 Agent：一个主 Agent，内部维护恐龙角色设定和故事状态。
- 图片生成：根据结构化 prompt 生成卡片。
- 内容安全：输入和输出都过安全规则，图片 prompt 也要限制。
- 存储：本地优先，家长确认后再保存。

## 故事状态模型

每次互动维护一个很小的状态对象：

```json
{
  "theme": "adventure",
  "scene": "彩虹森林",
  "activeDino": "小暴",
  "goal": "找到蓝色恐龙蛋",
  "turn": 3,
  "babyChoice": "走彩虹桥",
  "cardPrompt": {
    "dino": "小霸王龙",
    "color": "蓝色",
    "action": "举着星星",
    "object": "彩虹蛋"
  }
}
```

## 第一阶段里程碑

### P0：文本 Demo

- 无语音。
- 无图片。
- 能跑 3 只恐龙的短故事互动。
- 验证故事节奏和安全边界。

### P1：语音 Demo

- 加语音输入和 TTS。
- 宝宝可以听恐龙说话并回答。
- 家长能选择主题。

### P2：卡片 Demo

- 每局结束生成一张恐龙卡片。
- 家长可以保存或删除。

### P3：可试玩 MVP

- 移动端界面。
- 家长模式。
- 基础隐私策略。
- 10 到 20 个故事开场模板。

## 推荐下一步

先不要马上做大而全应用。下一步建议做一个 30 分钟内能试玩的 P0：

- 一个页面。
- 三只恐龙。
- 一个主题：彩虹森林冒险。
- 每轮 5 到 8 句。
- 每次给宝宝两个选择。
- 最后生成一段卡片 prompt。

只要这个文本 Demo 玩起来有趣，再加语音和图片就会很自然。
